# 测试说明

## 测试目标

本项目的测试目标是保障 2048 核心逻辑稳定，覆盖正常移动、合并、胜负判定、撤销、不同棋盘尺寸和随机生成方块等关键行为。

## 技术栈

- 测试框架：Mocha
- 断言库：Chai
- 浏览器环境模拟：JSDOM
- 覆盖率工具：NYC
- CI 平台：GitHub Actions

## 本地运行

```bash
npm install
npm test
```

## 覆盖率运行

```bash
npm run test:coverage
```

当前质量门禁：

- 行覆盖率不低于 70%
- 函数覆盖率不低于 70%
- 分支覆盖率不低于 60%

## 已覆盖的核心场景

- 初始化 4×4 棋盘并生成 2 个初始方块
- `slideRow()` 的相邻合并、隔空合并、连续三个相同值、四个相同值、全零行
- 左、右、上、下移动
- 到达 2048 的胜利判定
- 满盘不可移动的失败判定
- 满盘但仍可合并时不失败
- 有空格 / 无空格时的 `spawnTile()` 行为
- 有快照 / 无快照时的撤销行为
- 3×3 和 5×5 棋盘尺寸切换
